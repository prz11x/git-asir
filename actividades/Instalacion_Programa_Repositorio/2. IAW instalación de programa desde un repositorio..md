
# Instalación y prueba de FossFLOW en Ubuntu

## 1. Descripción del proyecto

FossFLOW es una aplicación web de código abierto orientada a la creación de diagramas isométricos. Permite diseñar visualmente infraestructuras, flujos o arquitecturas mediante elementos gráficos, facilitando la organización y representación de sistemas complejos.

En esta práctica se ha instalado FossFLOW en un servidor Ubuntu, se han probado sus funcionalidades básicas y se ha documentado todo el proceso paso a paso.

---

## 2. Requisitos del sistema

El servidor debe cumplir los siguientes requisitos:

- Sistema operativo **Ubuntu 20.04, 22.04 o superior**
- Acceso a la terminal con permisos sudo
- **Git**
- **Node.js (versión LTS) y npm**
- (Opcional) Docker y Docker Compose
- Navegador web local o acceso mediante túnel SSH

---

## 3. Actualización del sistema

Antes de instalar cualquier software, se actualiza el sistema para evitar problemas de dependencias:

```bash
sudo apt update && sudo apt upgrade -y
````

## 4. Instalación de Git

Git es necesario para clonar el repositorio del proyecto:

`sudo apt install git -y`

Verificación:

`git --version`

---

## 5. Instalación de Node.js LTS y npm

Se recomienda usar NodeSource para instalar la versión LTS de Node.js:

`curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash - 

`sudo apt install -y nodejs`

Verificación de la instalación:

`node -v npm -v`

---

## 6. Clonado del repositorio FossFLOW

Se descarga el código fuente desde GitHub y se accede al directorio del proyecto:

`git clone https://github.com/stan-smith/FossFLOW.git 

`cd FossFLOW`

---

## 7. Instalación de dependencias del proyecto

Se instalan todas las dependencias necesarias definidas en el proyecto:

`npm install`

Este proceso puede tardar varios minutos dependiendo del servidor.

---

## 8. Construcción del proyecto

Antes de ejecutar la aplicación, es necesario compilar la librería:

`npm run build:lib`

---

## 9. Ejecución de la aplicación

Para ejecutar FossFLOW en modo desarrollo:

`npm run dev`

Por defecto, la aplicación queda disponible en:

`http://localhost:3000`

---

## 10. Acceso desde el navegador local (túnel SSH)

Si el servidor no tiene entorno gráfico, se utiliza un túnel SSH desde el equipo local:

`ssh -L 3000:localhost:3000 usuario@IP_DEL_SERVIDOR`

`PS C:\Users\pablo> ssh -L 3000:localhost:3000 rui11@10.188.193.76`

Después, se accede desde el navegador local a:

`http://localhost:3000`

---

## 11. Pruebas realizadas en la aplicación

Una vez en la interfaz web de FossFLOW, se realizaron las siguientes pruebas:

- Creación de un diagrama isométrico nuevo.
    
- Añadido de varios elementos al diagrama.
    
- Guardado o exportación del proyecto.
    
- Importación del proyecto guardado para comprobar que los datos se conservan correctamente.
    

Estas pruebas permiten verificar el correcto funcionamiento básico de la aplicación.

---

## 12. Problemas encontrados y soluciones

- **Puerto no accesible**: se solucionó mediante túnel SSH.
